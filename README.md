# Снятие лексической неоднозначности

## Техническое задание

Реализовать систему, которая:

определяет наиболее подходящее значение многозначного слова в контексте (word sense disambiguation, WSD — снятие лексической неоднозначности);
извлекает и выводит семантические отношения (синонимы, гиперонимы, гипонимы) для выбранного значения.

Замечание: слово, для которого Вы выводите семантические отношения отмечаем сами. Например:  
Я буду сдавать лабораторные работы вовремя и получу хорошую оценку за экзамен.  
Отмеченное слово “оценка” . Для него решаем WSD задачу.

Для решения задачи необходимо использовать тезаурус WordNet или его аналог (например, RuWordNet, РуТез и др.), а также реализовать алгоритм Леска.

Использование готовых реализаций WSD запрещено.

## Решение

Взят ruwordnet - это тезаурус для русского языка, систематизированный словарь, отображающий семантические связи (синонимы, антонимы, иерархию понятий)

База RuWordNet была скачана в виде файла ruwordnet.db из репозитория https://github.com/avidale/python-ruwordnet

Для начала сделана подготовка текста, которая берет из него слова и их лемматизирует, а потом создает из результатов множество.

Затем сделана функция, которая расширяет словарь путем взятия синонимов, а также гиперонимов и гипонимов 1ого уровня для каждого слова.

Алгоритм Леска после препоцессинга и расширения берет текст для выбранного слова из определения и слов аналогичных тем, что получаются из расширения, а после выбирает лучший вариант слова путем сопоставления его с контекстом по соотношению совпавших слов ко всем, что связаны с вариантом.  
Такие модификации тривиального алгоритма увеличивают время обработки, но зато повышают количество совпадений и качество WSD даже с маленьким контекстом.

Также дополнительно сделан вывод синонимов, гиперомонимов и гипонимов для выбранного значения слова.

При развитии проекта также можно добавить введение метрики, которая будет оценивать реализацию на предобученном датасете.

После самопроверки выяснилось, что для тех слов, у которых нет определения, а только синонимы и т.п., алгоритм выдает score 1. Для исправления этого из контекста удаляется само слово. Также для дучшей иллюстративности были оставлены отладочные сообщения и дополнительные примеры и анализом.

## Результат

Реализован алгоритм Леска для определения значения слова в контексте.
Для повышения точности использовалась лемматизация слов и расширение контекста и сигнатуры.
После выбора значения из RuWordNet извлекаются его семантические связи: синонимы, гиперонимы и гипонимы.

В алгоритме не хватает дополнительных связей для лучшей обработки. Это видно по примеру с ключом и дверью. В сигнатуре для ключа есть замок, который является частью двери, поэтому, если смотреть пример с ключом и замком, то алгоритм отрабатывает правильно, но с дверью этого не происходит, так как ее самой в сигнатуре нет. Но, если решать эту проблему, алгоритм будет работать дольше.

Также примечателен пример с ручкой. На нее действует контекст из двух слов, которые тянут ее к разным определениям. На этом можно сделать предположение о том, что в таких случаях слово приобретет более общее значение. То есть конкрертно, да, в примере явно указана письменная ручка, но так как говориться о взятии, то алгоритм выбирает значение ручки для держания.

Но уже в такой реализации есть слова, которые определяются правильно. Например, такие как оценка и звезда.
